Executive Overview

Metricify is an intelligent festival planning application that combines Spotify listening data with EDM festival lineups to generate personalized, conflict-free itineraries. Built with a DevSecOps-first approach, the platform prioritizes security, scalability, and user experience.

Core Mission

Transform festival-goer experience through data-driven personalization and intelligent scheduling

Target Users

EDM enthusiasts attending multiple festivals who want optimized, personalized schedules

Security First

Enterprise-grade security with OAuth 2.0 PKCE, encrypted sessions, and automated scanning

API Endpoints

Authentication Endpoints
POST /api/auth/spotify/login

Initiates Spotify OAuth flow with PKCE

GET /api/auth/spotify/callback

Handles OAuth callback and exchanges authorization code for tokens

POST /api/auth/logout

Invalidates session and clears authentication cookies

Festival Data Endpoints
GET /api/festivals

Returns list of upcoming EDM festivals with filtering options

GET /api/festivals/:id/lineup

Retrieves complete lineup for a specific festival

Itinerary Endpoints
POST /api/itinerary/generate

Generates personalized itinerary based on user's Spotify data

GET /api/itinerary/:festivalId

Retrieves saved itinerary for a specific festival

PUT /api/itinerary/:id

Updates user's itinerary preferences

Security Architecture

Security-First Approach

All security measures are implemented following OWASP guidelines and industry best practices for authentication, data protection, and secure communication.

Security Layer Implementation Purpose
Authentication OAuth 2.0 with PKCE Prevent authorization code interception attacks
Session Management HTTP-only, Secure, SameSite cookies Prevent XSS and CSRF attacks
Token Storage AES-256 encryption at rest Protect stored OAuth tokens
Secrets Management AWS Secrets Manager/HashiCorp Vault Centralized, audited secret access
API Security Rate limiting, CORS, input validation Prevent abuse and injection attacks
Dependency Scanning Snyk/Trivy automated scans Identify vulnerable dependencies
Transport Security TLS 1.3, HSTS headers Encrypt all data in transit
